#### Why are PRs important in a team workflow?
Pull requests allow you to review changes before integrating them into the main code, facilitating error detection and aiding team collaboration. They also make it easier to maintain quality code that is separated and well-versioned to await reviews and approvals.

#### What makes a well-structured PR?
*It has a very clear title about what changes and a description of the changes made.
* It's small and focused on a single objective.
* It includes references to related issues.
* It facilitates review by having clear commits.

#### What did you learn from reviewing an open-source PR?
How large projects review changes in great detail, including code style, impact, and other components.

In addition to the importance of clarifying what is being worked on when making significant changes.
